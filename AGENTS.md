# AGENTS.md

## Цель

Помогать развивать текст о мышлении игрока в Tanks Blitz.

Не превращать текст в:

- сухой гайд;
- список советов;
- "10 способов нагибать";
- безличную инструкцию для новичков.

Главная идея проекта:

> Игрок проигрывает не из-за танка, а из-за решений.

## Канонический проект

- Рабочий git-репозиторий: `/home/qserj/Projects/ArtOfWar`.
- Основной текст: `ArtOfWar.md`.
- Очередь идей для будущего добавления: `Inbox.md`.
- Список задач по главам и доработкам: `Tasks.md`.
- Если рядом существуют внешние дубли файлов, не считать их источником истины.
- Не плодить новые копии основного текста без явного запроса.

## Карта файлов

- `ArtOfWar.md` - основной текст книги. Править его точечно, по главам и смысловым фрагментам.
- `Inbox.md` - входящая очередь идей. Сюда складывать мысли, которые пока не встроены в основной текст.
- `Tasks.md` - список задач. Сюда заносить главы на вычитку, улучшения и отдельные кандидаты на встраивание.
- `readme.md` - навигация по репозиторию для человека.
- `AGENTS.md` - рабочие правила для AI-редактуры.
- `.gitignore` - локальные служебные файлы, которые не должны попадать в git.

Файлы должны ссылаться друг на друга там, где это помогает навигации:

- `readme.md` объясняет, где основной текст, где inbox и где правила;
- `Inbox.md` напоминает, что после переноса фрагмент удаляется;
- `Tasks.md` показывает, что именно нужно вычитать, улучшить или встроить;
- `AGENTS.md` фиксирует рабочий процесс и авторский голос.

## Основной принцип

Текст должен объяснять:

> как думать, а не что нажимать.

Ценность текста не в том, чтобы дать игроку команду, а в том, чтобы показать причину:

- почему это решение работает;
- когда оно перестает работать;
- какую ошибку мышления оно исправляет;
- что игрок должен заметить в бою.

## Стиль

- живой, разговорный, но не хаотичный;
- короткие абзацы;
- ясные формулировки;
- примеры из боя;
- без воды;
- без мотивационных фраз;
- без клише вроде "играйте аккуратно";
- без лишней теории без привязки к игре.

Авторский голос важен.
Не сглаживать текст до нейтрального гайда.

## Рабочий процесс

После первичного приведения в порядок работать точечно, по главам.

Обычный цикл:

1. Понять новую мысль пользователя.
2. Определить подходящую главу.
3. Внедрить мысль в нужный фрагмент.
4. Убрать повторы, если они появились.
5. Сохранить структуру и голос текста.

Не переписывать весь документ ради мелкой правки.
Не переносить куски между главами без причины.
Если новая мысль пока не имеет места, временно предложить отдельный раздел-кандидат, а не ломать структуру.
Если мысль сохранена в `Inbox.md`, после переноса в `ArtOfWar.md` удалить ее из `Inbox.md`.
Если появилась новая задача по главе, добавить ее в `Tasks.md`.
После выполнения задачи отметить ее в `Tasks.md` или удалить, если она больше не нужна.

## Текущая структура текста

Основной текст собран вокруг крупных глав:

- вступление;
- кто автор;
- время и окна;
- позиция и геометрия;
- внимание и фокус;
- цена решения;
- обучение и разбор;
- короткая памятка;
- финальная мысль.

При добавлении новых идей сначала искать место внутри этой структуры.
Новую главу добавлять только если мысль не помещается в существующие.

## Приоритеты редактуры

1. Усилить смысл.
2. Улучшить ясность.
3. Убрать лишнее.
4. Добавить конкретику.
5. При необходимости дать короткий пример из боя.

Хорошее улучшение обычно делает мысль точнее, а не длиннее.

## Термины

Использовать и пояснять в контексте:

- КД (перезарядка);
- окно;
- фокус;
- позиция;
- угол;
- hulldown;
- размен;
- контроль внимания.

Термины не должны висеть как словарь.
Они должны работать внутри игровых ситуаций.

## Чего избегать

- повторения уже написанного;
- огульного переписывания всего документа;
- усложнения ради "умности";
- длинных списков без объяснений;
- очевидных советов без причины;
- пафоса и самоутверждения;
- превращения авторской главы в рекламную биографию.

## Что делать

- углублять отдельные главы;
- добавлять реальные игровые примеры;
- усиливать формулировки;
- упрощать сложные мысли;
- связывать практику с причиной;
- сохранять ощущение, что текст написан живым игроком, который разбирает решения.
